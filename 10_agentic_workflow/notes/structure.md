# Course Syllabus: Mastering Agentic Workflows

**From Foundational Concepts to Advanced Deployment and Scaling**

## Course Overview

This course is a practical, business-focused guide to building and deploying Agentic Workflows for real-world automations. Emphasizing a hands-on "build to learn" philosophy, students will move beyond using AI as a glorified copy-paste tool and learn to engineer autonomous, natural language-driven systems that execute complex, multi-step tasks.

---

### Module 1: Foundations of Agentic AI

*Understanding the landscape and the core differences between traditional logic and agentic behavior.*

* **The AI Landscape:** * **AI Overhang:** Understanding the gap between public perception of AI capabilities and its true potential.
* **The Arbitrage Window:** Capitalizing on the knowledge disparity in the market to generate revenue before the window closes.


* **Agentic Workflows vs. Traditional Automation:**
* Moving from rigid, rule-based automation (which requires months to master) to flexible, natural language-driven approaches.
* *Case Study:* Handling complex lead scraping and email outreach via natural language.



### Module 2: Core Architecture (The PTMRO Framework)

*The fundamental building blocks that allow an AI agent to operate autonomously.*

* **Planning:** Breaking down high-level objectives into executable, logical steps.
* **Tools:** Defining the actions an agent can take in the real world (e.g., API calls, web scraping).
* **Memory:** Managing how agents retain and recall information (Short-term, Mid-term, and Long-term).
* **Reflection:** Engineering the agent's ability to self-correct, evaluate outputs, and improve performance.
* **Orchestration:** The overarching coordination and management of the agent's sequence of actions.

### Module 3: Developer Environments & Workspace Setup

*Setting up the technical foundation and tools needed to build and interact with agents.*

* **Evolution of Interfaces:** The shift from Graphical User Interfaces (GUIs) to Text-Based Interfaces.
* **Integrated Development Environments (IDEs):** * Introduction to AI-first IDEs (VS Code, Cursor, Anti-Gravity).
* Navigating the three main panels: File Explorer, Editor Panel, and Agent Chat Panel.


* **Project & Agent Workspaces:**
* Structuring folders for maximum efficiency.
* `directives/`: Natural language standard operating procedures (SOPs) written in Markdown (e.g., `scrape_leads.md`).
* `execution/`: Python scripts and executable components.
* *System Files:* Understanding `.prompts`, `.venv`, `.claude`, and letting AI manage background configurations.



### Module 4: Practical Workflow Building

*Hands-on development of real-world business automations.*

* **The DOE Framework (Directive Orchestration Execution):** * Setting "guardrails" for agents using Directives (supercharged SOPs), explicit Tools, strict "Definition of Done" criteria, and Graceful Failure protocols.
* **Project 1: Automated Proposal Generation:**
* Feeding sales transcripts to an agent to extract problems, solutions, and pricing, followed by automated email follow-ups.


* **Project 2: ClickUp CRM Manager:**
* Building a tool-agnostic workflow to create leads, generate meeting URLs, and update pipeline statuses via API.


* **Project 3: LinkedIn Lead Scraper:**
* Processing Sales Navigator URLs, extracting leads, and handling user inputs.


* **Claude Skills & Portability:** Standardizing workflows into reusable packages (instructions, scripts, and `skill.md` files).

### Module 5: Context, Memory, and Efficiency

*Optimizing how models handle data and how developers interact with the AI.*

* **Managing the Context Window:**
* Understanding tokens and the Model Context Protocol (MCP).
* **Compaction:** Summarizing conversation history to reduce token usage while retaining vital context.


* **Maximizing Developer Productivity:**
* *Input Methods:* Using speech-to-text for commands rather than typing.
* *Prompting:* Specificity in requests (the "restaurant order" analogy).
* *Context Feeding:* Pasting API documentation directly into the IDE to reduce hallucination.
* *Monitoring:* Watching workflows run to understand agent reasoning and setting up completion hooks (e.g., audio notifications).



### Module 6: Deployment, Scaling, & Multi-Agent Systems

*Taking workflows out of the local environment and running them at scale.*

* **Cloud & Webhook Deployment:**
* Transitioning to autonomous, schedule-based, or event-driven operations.
* Setting up webhook listeners to trigger workflows instantly (e.g., acting on a newly captured lead).


* **Scaling via Parallelization:** * Running multiple agents concurrently for high-volume tasks (e.g., content scraping, community management, and competitor analysis simultaneously).
* **Advanced Architecture: Sub-Agents:**
* Delegating specific, heavy-lifting tasks to isolated sub-agents to prevent context window bloat in the main orchestrator agent.
* Testing, debugging, and refining sub-agent loops (managing rate limits, API calls, and string matching).
